What is Accessibility

According to MDN web docs,

Accessibility is the practice of making your websites usable by as many people as possible.


Accessibility is used to give everyone the same opportunities in accessing a website, no matter what their ability or circumstances.

Improving accessibility on your site also provides you with a lot of benefits as well. Crucially, it immediately expands your potential audience.

How To Implement Accessibility In Website

There are things to consider from the start of creating website in order to implement accessibility:

  • Test the website early and often for accessibility aspects.
  • Always bear in mind that a lot of accessibility best practices benefit everyone, not just users with disabilities.

Good practices that improves accessibility:

  • Use semantic HTML
  • Make sure all content is easily accessible. This can be reached by using ARIA.
  • Choose fonts and colors carefully.
  • Avoid automatic media and navigation.
Accessibility and HTML

Visually-impaired or blind users access web pages with the help of screen readers. The screen reader uses a wide variety of keyboard commands to read the HTML's contents out loud, navigating web pages, and take actions such as clicking on a link, typing in an input field, or submitting a form. Here are some things that you can do to provide accessibility with HTML.



Semantic Elements

Semantic is a linguistic study of meaning. In HTML it is the tags we use to mark a document. Some examples of semantic elements in HTML:

  • <strong> tag is used to mark bold letters / words.
  • <h1> - <h6> tag is used to mark headers.
  • <p> tag is used to mark a paragraph.
  • <code> tag is used to mark code reference.
  • <header> tag represents introductory content, typically a group of introductory or navigational aids. It may contain some heading elements but also a logo, a search form, an author name, and other elements.

You can find other semantic elements in HTML here.



ARIA

Accessible Rich Internet Applications (ARIA) is a set of attributes that define ways to make web content and web applications (especially those developed with JavaScript) more accessible to people with disabilities.

ARIA is needed to supplement HTML so that interactions and widgets commonly used in applications can be passed to Assistive Technologies when there is not otherwise a mechanism. But there are also rules for using ARIA in HTML.

Things That You Should Know

Other things that can improve accessibility in website:

  1. Provide alternative text within <img> tag. For example: 
                      <img src="logo.png" alt="company logo">
  2. Give structure to your HTML code. For example: 
                      <header>
                  <section>
                  <article>
                  <aside>
                  <footer>
  3. Choose your colors carefully. For example:

    Lorem ipsum dolor sit amet consectetur adipisicing elit! This is not easy to read. Instead, a clearer contrast color would be much better. Lorem ipsum dolor sit amet consectetur adipisicing elit!

  4. Create <label> for every <input> in your form.
